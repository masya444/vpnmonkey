# VPN Monkey — бот для продажи VPN-доступа

## Что внутри
- `config.py` — все настройки, берутся из переменных окружения
- `database.py` — SQLite: пользователи, подписки, рефералы, платежи
- `panel.py` — интеграция с панелью 3x-ui (создание VPN-ключей на сервере)
- `subscription.py` — веб-сервер, отдает subscription-ссылку для v2rayNG/Hiddify/Happ
- `main.py` — сам бот (aiogram)

## Как это работает
1. Пользователь жмет /start → получает пробный доступ на N дней.
2. Бот создает клиента на твоем VPN-сервере через API 3x-ui.
3. Пользователь получает не "ключ", а **ссылку подписки** — вставляет её один раз
   в приложение (v2rayNG / Hiddify / Happ), дальше всё обновляется само.
4. Оплата тарифов сейчас **ручная**: бот присылает тебе (админу) уведомление
   с командой `/confirm <user_id> <дни> <сумма>` — вводишь её, когда видишь перевод.
   Это временное решение, пока не подключишь автоматическую оплату (ЮKassa/CryptoBot).

## Что нужно перед запуском

### 1. VPS для VPN-сервера
Купи VPS в Нидерландах/Германии/Финляндии (от ~5-10$/мес). На него ставится:
```
bash <(curl -Ls https://raw.githubusercontent.com/MHSanaei/3x-ui/master/install.sh)
```
(команда актуальна на момент написания — проверь README проекта 3x-ui на GitHub,
могла обновиться)

В панели 3x-ui создай один inbound с протоколом **VLESS + Reality**.

### 2. Переменные окружения
Создай файл `.env` (или пропиши переменные в настройках Railway/сервера):
```
BOT_TOKEN=токен_от_BotFather
ADMIN_ID=твой_telegram_id
SERVER1_PANEL_URL=https://IP-сервера:2053
SERVER1_PANEL_USER=admin
SERVER1_PANEL_PASS=пароль_от_панели
SERVER1_INBOUND_ID=1
SERVER1_PUBLIC_HOST=IP-сервера
SUB_PUBLIC_BASE_URL=http://IP-сервера-с-ботом:8080
```

**Важно:** `panel.py` строит `vless://` ссылку по шаблону в функции `build_vless_link`.
Параметры (`type`, `security`, `pbk`, `sid` для Reality) зависят от твоих точных настроек
inbound-а. Сделай так:
1. В 3x-ui создай тестового клиента руками.
2. Нажми "Export"/скопировать ссылку рядом с ним.
3. Скопируй оттуда все параметры после `?` и вставь в `build_vless_link`,
   заменив только UUID и host на переменные — так ссылка будет 100% совпадать
   с твоей реальной настройкой сервера.

### 3. Установка и запуск
```
pip install -r requirements.txt --break-system-packages
python main.py
```

## Чек-лист безопасности (обязательно перед тем как звать реальных людей)
- [ ] Порт панели 3x-ui сменен со стандартного на случайный
- [ ] Файрвол (ufw) на VPS: доступ к порту панели только с твоего IP
- [ ] `.env` добавлен в `.gitignore`, никогда не запушен в открытый GitHub
- [ ] Пароль от панели — длинный и уникальный, не тот же, что от почты/телеги
- [ ] Настроен бэкап `x-ui.db` хотя бы раз в неделю (например, cron + отправка себе в облако)
- [ ] Обновляешь 3x-ui, когда выходят новые версии — там закрывают найденные уязвимости

## Разработчику на будущее
Схема "субскрипшн-ссылка вместо разового ключа" — это ровно то, как устроены топовые
VPN-боты рынка (Hit VPN и подобные), просто без необходимости писать свое нативное
приложение — ты используешь готовые (v2rayNG/Hiddify/Happ), а свою ссылку встраиваешь в них.
